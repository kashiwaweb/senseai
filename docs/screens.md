## 2. ユーザーフロー

### フェーズ①: 「型」を作る（1ヶ月目で完成）

```
1. ログイン
   ↓
2. /staff でスーパースタッフを登録
   ↓
3. /upload で入力モードを選択
   ├─ 音声モード: mp3/m4a/wav を選択 → Groq Whisper で自動文字起こし
   └─ テキストモード: 起こし済みテキストを貼り付け
   ↓
4. 種別（customer/instruction/philosophy）を指定して保存 → /sessions に追加
   ↓ (5〜10セッション貯まったら)
5. /profiles で「型を生成」ボタン → 30〜60秒待つ
   ↓
6. /profiles/:id で7ステップ・魔法の言葉を確認
   ↓
7. オーナーが内容をレビュー・必要なら再生成
```

### フェーズ②: 一般スタッフを診断（2ヶ月目で完成）

```
1. ログイン
   ↓
2. /staff で一般スタッフを登録
   ↓
3. /upload で入力モード（音声/テキスト）を選択し、スタッフを指定
   ↓
4. 音声時は文字起こし、テキスト時はそのまま保存
   ↓
5. /sessions/:id で「診断する」ボタン
   ↓
6. /diagnoses/:id へ自動遷移
   ↓
7. 7ステップのレーダーチャート・改善例を確認
```

## 3. 画面詳細

### 3.1 ログイン (`/login`)

- 共有パスワード入力欄1つ
- 「ログイン」ボタン
- エラー時は「パスワードが違います」のみ表示

### 3.2 ダッシュボード (`/`)

- サマリカード4つ:
  - スタッフ数 / セッション数 / 「型」バージョン / 診断回数
- 最近のアクティビティリスト（直近10件）
- CTA ボタン: 「アップロード」「型を見る」「診断結果を見る」

### 3.3 スタッフ一覧 (`/staff`)

- テーブル: 名前 / 役割（バッジ表示: super=金, trainee=青）/ セッション数 / 最終診断日
- 「+ スタッフ追加」ボタン → モーダル（名前 / 役割選択）

### 3.4 スタッフ詳細 (`/staff/:id`)

- プロフィール（編集可能）
- セッション一覧（種別・録音日・長さ）
- 一般スタッフの場合のみ:診断履歴のミニチャート

### 3.5 セッションアップロード (`/upload`)

- スタッフ選択（プルダウン）
- セッション種別（ラジオ: 接客 / 指導 / 心掛けスピーチ）
  - スーパースタッフ選択時のみ「指導」「心掛けスピーチ」が有効
- **入力モード切替（タブ）: 音声 / テキスト**
  - **音声モード**:
    - 音声ファイル選択（drag&drop 対応, mp3/m4a/wav, 最大100MB）
    - アップロード進捗バー
    - 完了後 Groq Whisper で文字起こし → 結果ペインに表示・編集可能（誤認識の修正用）
  - **テキストモード**:
    - テキストエリアに書き起こし済みの会話を貼り付け
    - （任意）録音日・録音時間を手入力
- 「保存」ボタンで Session として確定

### 3.6 セッション一覧 (`/sessions`)

- フィルタ: スタッフ / 種別 / 期間
- テーブル: 録音日 / スタッフ / 種別 / 長さ / 文字起こし冒頭 / 操作（詳細/削除）

### 3.7 セッション詳細 (`/sessions/:id`)

- 音声プレイヤー
- 文字起こし全文（編集モード切替可能）
- 一般スタッフのセッションの場合は「診断する」ボタン
- スーパースタッフのセッション一覧から「型生成」もここから可能

### 3.8 「型」一覧 (`/profiles`)

- バージョンごとのカード一覧（v1, v2, ...）
- 各カード: 生成日 / 元セッション数 / モデル名 / 「詳細」ボタン
- 「+ 新規生成」ボタン → 元にするセッションを選ぶモーダル → 生成開始

### 3.9 「型」詳細 (`/profiles/:id`)

- ヘッダー: バージョン・生成日・元セッション一覧・LLMモデル・プロンプトバージョン
- セクション1: 全体特徴（overview / core_values / tone）
- セクション2: 7ステップ（アコーディオン展開）
  - 各ステップ:
    - description
    - key_behaviors リスト
    - magic_words テーブル（フレーズ / 意図 / 場面）
    - common_pitfalls
- 「再生成」ボタン

### 3.10 診断結果一覧 (`/diagnoses`)

- フィルタ: スタッフ / 期間
- テーブル: 診断日 / スタッフ / セッション / 平均スコア / 詳細

### 3.11 診断結果詳細 (`/diagnoses/:id`)

- ヘッダー: スタッフ名・セッション情報・対象「型」のバージョン
- 全体サマリ（overall_summary）+ 励ましメッセージ（encouragement_message, 大きめ表示）
- **7ステップのレーダーチャート**（中央に大きく）
- 各ステップ詳細（カード並び）:
  - スコア（5段階バッジ）
  - 強み: 観察＋実際の発話引用
  - 伸びしろ: 観察＋次にやってみるとよいこと
  - 改善例テーブル: 「あなたの言葉」→「トップならこう言う」+ 理由
- 印刷可能ビュー（スタッフへの面談で紙で渡せるように）

### 3.12 設定 (`/settings`)

- 店舗名
- LLMプロバイダー切替（gemini / claude）
- パスワード変更
- データエクスポート（将来）

## 4. デザイン方針

- **トーン**: 優しい・温かい・専門的すぎない
- **カラー**: ベージュ／淡いゴールドを基調（サロン業界に親和）
- **フォント**: Noto Sans JP（読みやすさ優先）
- **「ダメ出し」表現NG**: スコアが低くても「これから伸ばす」表現に統一
- **モバイル**: スマホブラウザで閲覧できれば良い（ネイティブアプリは作らない）
- **CSS**: Tailwind CSS（Nuxt UI でも可）
