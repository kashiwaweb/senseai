# senseai 要件定義書

最終更新: 2026-05-01

## 1. プロジェクト目的

トップスタッフ（スーパースタッフ）の「接客のSense（感覚）」をAIで言語化・体系化し、
サロン全員が再現可能な「技術」に変換する。

> キャッチコピー: トップの"感覚"を、全員の"技術"にする。

## 2. ターゲットと利害関係者

| 役割         | 説明                                     |
| ------------ | ---------------------------------------- |
| 主たる利用者 | エステサロンのオーナー（PoC: 1店舗）     |
| 育成基準     | スーパースタッフ（録音・スピーチを提供） |
| 受益者       | 一般スタッフ（診断対象）                 |
| 間接受益者   | お客様（接客品質向上の恩恵）             |

## 3. 解決する課題

1. 接客・おもてなしのスキルは「感覚」に依存しており、教育が困難
2. 教える先輩によってアドバイスの基準・ニュアンスが統一されない
3. 若手スタッフが「具体的にどう改善すれば良いか」迷う
4. トップスタッフの教育負担が大きい

## 4. ソリューション概要

2フェーズ構成:

### フェーズ① 基準づくり（1ヶ月目）

スーパースタッフの3種類の音声を蓄積し、AIが「7ステップの型」「魔法の言葉」として言語化。

- `customer`: お客様との接客録音
- `instruction`: スタッフへの指導音声
- `philosophy`: 心掛け・接客哲学のスピーチ

### フェーズ② 個別診断（2ヶ月目）

一般スタッフの接客録音をアップロード→AIが「型」と比較→
7ステップごとの強み・伸びしろ・「トップならこう言う」改善例を生成。

## 5. スコープ

### PoC で含むもの

- Webアプリ（PC + スマホブラウザ閲覧可）
- 単一店舗・単一ログイン（共有パスワード認証）
- スタッフ管理（追加・役割設定）
- 音声アップロード（mp3 / m4a / wav）
- 自動文字起こし
- スーパースタッフの「型」抽出と表示
- 一般スタッフの診断結果表示（7ステップ・レーダーチャート・改善例）

### PoC で含まないもの（将来拡張）

- マルチテナントSaaS化
- ネイティブモバイルアプリ
- リアルタイム録音機能
- 動画分析・表情分析
- 顧客同意管理システム
- 課金・サブスク機能
- スタッフごとのログイン
- 外部システム連携

## 6. 成功指標（KPI）

PoC段階の合格基準:

- 「型」抽出結果について、サロンオーナーから「自店舗のトップの良さを的確に表現している」と評価される
- 診断結果について、対象スタッフ本人から「具体的で実践しやすい」と評価される
- スケジュール通り 1ヶ月目末にスーパースタッフ機能、2ヶ月目末に一般スタッフ機能が公開される

## 7. 制約条件

| 項目         | 内容                                                                      |
| ------------ | ------------------------------------------------------------------------- |
| 予算         | 月額 ¥0〜¥1,400 程度を目標（PoC期）                                       |
| プライバシー | 顧客の会話を扱うため、PoC期間中は店舗内利用に限定。匿名化方針は別途定義。 |
| 言語         | 日本語のみ                                                                |
| 開発体制     | Yagiさん + Claude Code（2人体制）                                         |
| ホスティング | Vercel 無料サブドメイン                                                   |

## 8. 用語定義

| 用語             | 定義                                                     |
| ---------------- | -------------------------------------------------------- |
| スーパースタッフ | 店舗の接客基準となるトップ実力者                         |
| 一般スタッフ     | 育成対象のスタッフ                                       |
| 型（TopProfile） | スーパースタッフの接客パターンをAIが構造化した基準データ |
| 7ステップ        | お客様の心が動く接客プロセス                             |
| 魔法の言葉       | スーパースタッフの効果的な定型フレーズ集                 |
| セッション       | 1回分の音声入力単位（接客 / 指導 / スピーチ）            |
| 診断             | 一般スタッフのセッションを「型」と比較した結果           |

## 9. ロードマップ（8週間）

### 1ヶ月目: スーパースタッフのAI

| 週  | 達成事項                                         | アウトプット                                                |
| --- | ------------------------------------------------ | ----------------------------------------------------------- |
| W1  | プロジェクト初期化／プロンプト設計実験           | Nuxt骨組み・手動テキスト入力でAIが7ステップ分析する最小動作 |
| W2  | 音声アップロード→Whisper文字起こし               | mp3/m4aをアップして自動で会話テキスト化                     |
| W3  | 複数セッションから「型」抽出ロジック             | 5〜10セッションを統合し型JSONを生成                         |
| W4  | スーパースタッフ「型」表示UI / PoC現場で初テスト | 公開・サロン側のフィードバック収集                          |

### 2ヶ月目: 一般スタッフ診断

| 週  | 達成事項                                         |
| --- | ------------------------------------------------ |
| W5  | 一般スタッフ用アップロード画面・複数スタッフ管理 |
| W6  | 比較ロジック（型 vs 一般スタッフ会話）の実装     |
| W7  | 診断結果UI（7ステップレーダーチャート・改善例）  |
| W8  | PoCサロンで本番運用テスト・調整・公開            |
